    1. Java エンジニアに知って欲しい RDB アンチパターン

    ポイント

    • データにロジックを埋め込むな
    • アプリには、データベースの中身がわからない
    • DB を停止すると、サービスが止まる
    • DB を変更すると、複数サービスの改修が必要
    • データは常に変化する
    • アプリと違って、DB の知識、数年では廃れない。
    • モニタリング大事。

    コメント

    • 当たり前なんだけど、どれも大事。
    • 特に、モニタリング。
    • 何が良くて、何が悪いのかが推測になってしまう。
